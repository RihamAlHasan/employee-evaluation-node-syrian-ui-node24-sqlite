# Employee Evaluation System - Node.js Conversion

هذه نسخة Node.js محوّلة من مشروع ASP.NET Core المرفق، مع الحفاظ على الكيانات الأساسية ومنطق الصلاحيات والتقييم والاعتماد والتظلم.

## التشغيل السريع داخل Codex

```bash
npm install
npm test
npm start
```

ثم افتح:

```text
http://localhost:3000
```

## قاعدة البيانات

الافتراضي هو SQLite عبر ملف محلي:

```env
DB_PROVIDER=sqlite
SQLITE_PATH=./data/employee-evaluation.sqlite
```

لأن بيئات Codex قد لا تحتوي SQL Server، التطبيق يحفظ الحالة في SQLite بشكل durable عبر جدول `AppState`، بينما يوجد مخطط SQL Server علائقي كامل في:

```text
sql/sqlserver-schema.sql
```

يمكن استخدام SQL Server لاحقاً بتطبيق Repository علائقي فوق نفس `service` domain layer؛ المنطق نفسه مفصول في `src/domain` ومغطى بالاختبارات.

## ما تم تحويله

- تسجيل الدخول بالرقم الوطني أو الرمز الوظيفي.
- أدوار النظام: Admin، مسؤول تقييم مركزي، مسؤول تقييم جهة، مدير دائرة، مدير المديرية، موظف.
- إدارة الموظفين والدورات والنماذج.
- نشر/إغلاق دورة التقييم.
- نماذج مهام وسلوكيات مع أوزان مختلفة حسب نوع النموذج.
- الموظف يقيم نفسه ومديره وزملاءه المسندين.
- المدير يقيم موظفيه، نفسه، ومدراء آخرين عند الإسناد.
- مدير المديرية يقيم مدراء الدوائر.
- منع تكرار إرسال نفس التقييم.
- احتساب نتيجة النموذج والنتيجة النهائية.
- إخفاء نتيجة الموظف قبل اعتماد مسؤول التقييم المركزي.
- تعديل/اعتماد النتيجة النهائية مع سرديات القوة والتحسين.
- سجل نشاط للعمليات الحساسة.
- API بسيط للفحص: `/api/health`, `/api/employees`, `/api/cycles`, `/api/templates`.

## بنية الملفات

```text
src/domain/        منطق الأعمال والاختبارات عليه
src/routes/        مسارات Express
src/views/         واجهات EJS عربية RTL
src/public/css/    تنسيق الهوية البصرية
sql/               مخططات SQLite وSQL Server
tests/             اختبارات Node --test
```

## ملاحظة مهمة

هذه النسخة دقيقة في منطق العمل الأساسي والصلاحيات، لكنها ليست نسخاً حرفياً لكل صفحات Razor القديمة. الهدف أن يستطيع Codex تشغيلها واختبارها وتطويرها بسرعة داخل بيئة Node.js، ثم يمكن لاحقاً توسيع Repository الخاص بـ SQL Server إن كانت بيئة التشغيل تدعم اتصال SQL Server حقيقي.


## إصلاح مهم لبيئة Windows / Node 24

تم حذف اعتماد `better-sqlite3` من هذه النسخة، لذلك لن تحتاجي إلى Visual Studio C++ Build Tools أثناء `npm install`.

تعمل قاعدة SQLite عبر وحدة Node المدمجة `node:sqlite`، ومكان الملف مضبوط من `.env`:

```env
DB_PROVIDER=sqlite
SQLITE_PATH=./data/employee-evaluation.sqlite
```

الأوامر المقترحة:

```bash
rmdir /s /q node_modules
del package-lock.json
npm install --no-audit --no-fund
npm test
npm start
```
